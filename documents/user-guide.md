# SynchroDeck ユーザーガイド

## SynchroDeck とは

AI が生成した SVG スライドを VS Code 上で視覚的に編集できる拡張機能です。  
テキストの書き換え・図形の移動・サイズ変更・色の変更を GUI で行い、編集内容は自動的に SVG ファイルに反映されます。

---

## インストールと初回セットアップ

### 1. リポジトリをクローン

```bash
git clone https://github.com/kmchord9/SynchroDeck.git
cd SynchroDeck
```

### 2. 依存パッケージをインストール

```bash
npm install
```

### 3. ビルド

```bash
npm run build
```

### 4. 拡張機能を起動

VS Code で SynchroDeck フォルダを開き、**F5** を押します。  
Extension Development Host ウィンドウが開きます。

---

## 基本的な使い方

### スライドを開いてプレビューする

1. `slides/` フォルダ内の SVG ファイル（例: `slide_01.svg`）をエディタで開きます
2. **Ctrl+Shift+P** → **SynchroDeck: スライドプレビューを開く** を実行します
3. 右側にプレビューパネルが開きます

SVG ファイルを編集するたびにプレビューがリアルタイムで更新されます。

---

## GUI 編集機能

### シェイプパネル

プレビュー右上の **◈ シェイプ** ボタンをクリックするとパネルが開きます。  
スライド内のすべての要素が一覧表示されます。

| アイコン | 要素の種類 |
|----------|-----------|
| **T** | テキスト（foreignObject） |
| 塗り色の四角 | 矩形（rect） |
| 塗り色の円 | 円（circle） |

---

### テキストを編集する

**方法 1: シェイプパネルから**

1. シェイプパネルを開く
2. 編集したいテキスト行（T アイコン）右端の **✏** ボタンをクリック
3. テキストが編集モードになります（枠が緑色の点線に変わります）
4. テキストを自由に書き換えます
5. **Esc** または枠の外をクリックして確定

**方法 2: プレビュー上でダブルクリック**

テキスト領域をダブルクリックすると編集モードに入ります。

> **注意**: 編集内容は確定時に自動的に SVG ファイルへ保存されます。

---

### 図形・テキストを移動する

1. シェイプパネルで要素をクリックして選択（または図形を直接クリック）
2. 選択枠（青い点線）が表示されます
3. 選択枠の内側をドラッグして移動

---

### 図形・テキストをリサイズする

1. 要素を選択します
2. 選択枠の角または辺に表示される **8 つの白いハンドル** をドラッグ

| ハンドル位置 | 操作 |
|-------------|------|
| 四隅 | 縦横同時にリサイズ |
| 上下の辺 | 高さのみ変更 |
| 左右の辺 | 幅のみ変更 |

---

### 図形の色を変更する

1. 図形（rect または circle）を選択します
2. 選択枠の上部にカラーピッカーが表示されます
3. カラーピッカーをクリックして色を選択

> テキスト（foreignObject）はカラーピッカー非対応です。テキストの色を変更する場合は SVG ファイルを直接編集してください。

---

## PDF エクスポート

1. **Ctrl+Shift+P** → **SynchroDeck: PDF にエクスポート** を実行
2. 保存先を指定
3. `slides/` フォルダ内のすべての SVG ファイルが 1 つの PDF にまとめられます

スライドの順番はファイル名のアルファベット順です（`slide_01.svg` → `slide_02.svg` → …）。

---

## スライドファイルの構成

### ファイルの場所

```
SynchroDeck/
└── slides/
    ├── slide_01.svg    1 枚目
    ├── slide_02.svg    2 枚目
    └── slide_03.svg    ...
```

### SVG の基本構造

すべてのスライドは **1280×720 px** で作成します。

```xml
<?xml version="1.0" encoding="UTF-8"?>
<svg xmlns="http://www.w3.org/2000/svg" width="1280" height="720" viewBox="0 0 1280 720">

  <!-- 背景 -->
  <rect x="0" y="0" width="1280" height="720" fill="#1a1a2e"/>

  <!-- テキスト（自動改行対応） -->
  <foreignObject x="100" y="200" width="800" height="200">
    <div xmlns="http://www.w3.org/1999/xhtml"
         style="font-size:48px;font-weight:bold;color:#ffffff;">
      スライドタイトル
    </div>
  </foreignObject>

</svg>
```

### 新しいスライドを追加する

1. 既存の SVG ファイルをコピーしてリネーム（例: `slide_03.svg`）
2. 内容を編集
3. プレビューで確認

---

## Git によるバージョン管理

SVG ファイルはテキスト形式のため、Git で差分管理できます。

```bash
# 変更を確認
git diff slides/slide_01.svg

# コミット
git add slides/
git commit -m "タイトルスライドを更新"
```

## トラブルシューティング

### プレビューが表示されない

- SVG ファイルがエディタで開かれているか確認してください
- `Ctrl+Shift+P` → `SynchroDeck: スライドプレビューを開く` を再実行してください

### シェイプパネルに要素が表示されない

- ファイルが保存されているか確認してください
- プレビューパネルを閉じて開き直してください

### F5 で起動後に変更が反映されない

ターミナルで手動ビルドしてから F5 を押してください：

```bash
npm run build
```

### PDF エクスポートでエラーが出る

初回は Puppeteer が Chromium をダウンロードします（数分かかる場合があります）。  
ネットワーク環境によっては失敗することがあります。その場合は再度 `npm install` を実行してください。
